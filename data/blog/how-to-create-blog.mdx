## üõ†Ô∏è Step-by-Step Guide

### 1. Write Your Blog Post in `.mdx` Format

All blog posts are written using [MDX](https://mdxjs.com/) ‚Äî Markdown with React components support.

Use this structure as a starting point for your front matter:

```yaml
---
title: 'Your Blog Title'
date: 'YYYY-MM-DD'
summary: 'A short summary of your blog'
tags:
  - tag1
  - tag2
draft: false
thumbnail: '/static/images/blog/thumbnails/your-image.jpg'
authors:
  - default
---
```

### 2. Fork the GitHub Repository

Go to the GitHub repo:
üëâ https://github.com/BiditG/PortfolioBidit

Click the "Fork" button to make your own copy of the repository.

![Fork the repo](/static/images/blog/picturesforblog/fork.png)

### 3. Add Your .mdx File

Navigate to the data/blog/ folder in your forked repo.

Create a new .mdx file (e.g., my-awesome-blog.mdx).

Paste your blog post content into it and save the file.

![Navigate to blog folder](/static/images/blog/picturesforblog/location.png)

### 4. Commit and Push Your Changes

After editing, commit your changes with a clear message (e.g., Add my blog post: My Awesome Blog).

Push the changes to your forked GitHub repo.

### 5. Create a Pull Request

Go back to the original repo: https://github.com/BiditG/PortfolioBidit

Click ‚ÄúCompare & pull request‚Äù

Add a short description and submit your PR to the main branch.
